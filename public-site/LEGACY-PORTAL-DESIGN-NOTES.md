# Legacy Portal Design Notes

Source: harvested from `Mave9055/cptsd-education-portal` README during WRH consolidation.

Purpose: preserve useful design and accessibility ideas from the earlier public education portal without importing an entire duplicate web application.

## Portal Concept

The legacy portal framed the work as a professional public education website for CPTSD, attachment trauma, nervous-system dysregulation, recovery, and trauma-informed learning.

For the current WRH system, the stronger public-facing frame is:

> A non-clinical public education portal that teaches survival patterns, nervous-system adaptation, shame loops, avoidance, addiction, relational repetition, and repair language through plain-language lectures and structured learning paths.

## Useful Features to Preserve

- lecture-based public education structure
- clean accessible interface
- learning objectives for each lecture
- discussion questions for each lecture
- facilitator notes for public education settings
- responsive desktop/mobile design
- readable typography
- generous spacing
- low-pressure navigation
- trauma-informed pacing

## Design Principles

### Grounded Accessibility

The site should feel serious, calm, and readable without looking clinical, sterile, or sensational.

### Content First

The page should prioritize the teaching material, not visual tricks.

### Low Activation

Avoid jarring animations, sudden motion, aggressive colors, or layouts that pressure the reader.

### Clear Entry Paths

Different users should know where to start:

- first-time reader
- participant
- facilitator
- partner organization
- procurement reviewer
- family/support person

### No Forced Disclosure

The site should never imply that users must share personal history to benefit from the material.

## Lecture Page Template

Each public lecture page could include:

1. title
2. one-sentence purpose
3. plain-language overview
4. key pattern
5. common misreading
6. WRH mechanism translation
7. reflection question
8. optional discussion question
9. facilitator note
10. next lecture link

## Accessibility Notes

Preserve these commitments from the legacy portal concept:

- high contrast text
- readable font sizes
- semantic structure
- keyboard navigation
- visible focus states
- mobile-friendly layout
- plain language
- no unnecessary visual overload

## Boundary Language

Every public site should make the WRH scope clear:

- not therapy
- not medical advice
- not legal advice
- not crisis care
- not a replacement for licensed professionals
- non-clinical education and pattern-recognition support

## Decision

The legacy portal has useful public education structure and design thinking.

The current system should preserve the design principles and lecture map, but avoid maintaining the old portal as a separate competing curriculum home unless it is intentionally rebuilt as a public WRH front door.
